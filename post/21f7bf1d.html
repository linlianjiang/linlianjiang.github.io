GRU(Gate Recurrent Unit)

Created2022-04-16|Updated2022-04-16|knowledge

1. GRU

  1. GRU（Gate Recurrent Unit）是循环神经网络（Recurrent Neural Network, RNN）的一种
  2. 和LSTM（Long-Short Term Memory）一样，也是为了解决长期记忆和反向传播中的梯度等问题而提出来的
  3. 实验中选择GRU是因为它的实验效果与LSTM相似，但是更易于计算
  4. 相比LSTM，使用GRU能够达到相当的效果，并且相比之下更容易进行训练，能够很大程度上提高训练效率，因此很多时候会更倾向于使用GRU

1.1 GRU的输入输出结构

  1. GRU的输入输出结构与普通的RNN是一样的

  2. 如上图所示：

    • 输入 $x^{t}$ 和上一个节点的 hidden state $h^{t-1}$ (hidden state 包含之前节点的相关信息)

    • 结合 $x^{t}$ 和 $h^{t-1}$ ， GRU会得到当前隐藏节点的输出 $y^{t}$ ，和传递给下一个节点的隐状态 $h^{t}$

1.2 GRU的内部结构

  1. 先通过上一个传输下来的状态 $h^{t-1}$ 和当前节点的输入 $x^{t}$ 来获取两个 gate 的状态：

    • $r$ ：控制 reset gate
    • $z$：控制 update gate
    • $\sigma$ ：sigmoid 函数

  2. 具体过程

    • 得到 gate 信号后，首先使用 reset gate 来得到 “reset” 后的数据 :

    • 将 $h^{t-1’}$ 与输入 $x^{t}$ 进行拼接 ， 通过一个 tanh激活函数 来将数据放缩到-1~1的范围内

    • 这里 $h^{‘}$ 主要是包含了当前输入的 $x^{t}$ 数据 ，有针对性对 $h^{‘}$ 添加到当前的隐藏状态，相当于 “记忆当前时刻的状态” 。 类似于LSTM的选择记忆阶段

  3. GRU最关键的一个步骤：”更新记忆“阶段

    • 在这个阶段，同时进行了遗忘了记忆两个步骤。使用先前得到的 update gate

    • 更新表达式 :

      gate 信号（z）的范围为0~1。gate 信号越接近1，代表”记忆“下来的数据越多；而越接近0则代表”遗忘“的越多

    • Pytorch里的GRU写法相比原版多了一个映射，相当于一个GRU变体，猜测是多加多这个映射能让整体实验效果提升较大

    • GRU很聪明的一点就在于，使用同一个 gate $z$ 就同时可以进行遗忘和选择记忆（LSTM需要多个 gate）

  4. 相关公式：

    • : 表示对原本 hidden state 的选择性“遗忘” ， 可以想象成 forget gate ，忘记 $h^{t-1}$ 维度中一些不重要的信息
    • ： 表示对包含当前节点信息的 $h^{‘}$ 进行选择性”记忆“ 。 与上面类似 ，这里 会忘记 $h^{‘}$ 维度中一些不重要的信息。 或者，这里我们更应当看做是对 $h^{‘}$ 维度中的某些信息进行选择
    • ： 结合上述，这一步的操作就是忘记传递下来的 中的某些维度信息，并加入当前节点输入的某些维度信息

  5. 可以看到，这里的遗忘 $z$ 和选择 是联动的。也就是说，对于传递进来的维度信息，我们会进行选择性遗忘，则遗忘了多少权重 $z$，我们就会使用包含当前输入的 $h^{‘}$ 中所对应的权重进行弥补 。以保持一种”恒定“状态
