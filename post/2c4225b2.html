20_ECCV_PT2PC Learning to Generate 3D Point Cloud Shapes from Part Tree Conditions

Created2022-02-28|Updated2022-03-12|paper learningPoint Cloud
paper code

0. Abstract

Title: PT2PC: Learning to Generate 3D Point Cloud Shapes from Part Tree Conditions

  1. “part-tree” to “point cloud” (PT2PC)，分解结构和几何因素
  2. 从符号零件树表示生成形状的三维点云几何体的新问题
  3. 还提出了一种新的结构方法来评估生成的形状点云是否满足零件树的条件

1. Introduction

  1. explicitly modeling part semantics and shape structures（显式地建模零件语义和形状结构）

  2. 生成三维点云形状与geometric variations conditioned （几何变化条件）下的structural shape descriptions（结构形状描述）

  3. 上图描述了输入和输出

  4. 将每个3D形状表示为零件的层次结构，遵循PartNet，其中每个零件节点有一个关联的语义标签，零件层次结构包括不同分割粒度的零件

  5. 抽象出具体的零件几何，形状结构可以由一个符号零件树定义，只有零件的语义和它们之间的关系(图1，左)

  6. 鉴于这种符号部分树条件，我提出了一种条件 conditional-GAN PT2PC来生成满足结构条件的各种三维点云形状(图1，右)

  7. 符号部分树条件是 PT2PC generator（生成器）和 discriminator（鉴别器）体系结构设计的核心

  8. generator 首先使用每个部件节点的语义和结构信息，以自底向上的方式沿着树层次结构编码部件树模板特性

  9. 然后，给定一个random noise vector（随机噪声向量），在根节点上捕获全局几何信息，沿着零件树以自顶向下的方式递归地传播这些几何信息到每个零件节点

  10. 最后的点云是通过聚合在表示其对应的 fine-grained 语义部分的每个叶节点上解码的点云生成的

  11. discriminator 首先在 leaf level 计算每个部分的特征，沿着 tree hierarchy 以 bottom-up 的方式传播信息，直到根节点，最后生成一个分数，判断生成的形状几何形状是否真实，形状结构是否满足输入条件

  12. 在PartNet 数据集的四个主要 semantic classes 上评估提出的模型。

  13. 为了证明 tree-structure architecture 对 generator 和 discriminator 的优点，比较了两个条件GAN基线

  14. 定量和定性结果都表明，无论在可见模板还是不可见模板条件下，我们的设计在整体形状质量、零件形状质量和形状多样性方面都具有明显的优势

  15. 此外，提出了一种新的分层零件实例分割方法，该方法能够将没有任何零件标签的输入点云分割成一个符号零件树

  16. Contributions：

    • 提出了 part-tree 条件下点云生成的新任务
    • 提出了 conditional GAN ，PT2PC，在给定符号部分树条件的情况下，生成真实和多样的点云形状
    • 提出了一种新的点云结构评价指标，用于评价生成的点云是否满足零件树条件

2. Related Works

综述了 3D generative shape modeling,、part-based shape modeling和 structure-conditioned content generation（基于结构条件的内容生成）等方面的相关工作

3D Generative Shape Modeling

Part-based Shape Modeling

使用PartNet 中定义的树层次结构，并提出一个新的任务PT2PC，在给定符号部分树条件的情况下生成点云形状

Conditional Content Generation

一个条件生成对抗网络，生成形状点云条件的输入符号部分树结构

3. Method

  1. conditional GAN（c-GAN）从给定的 symbolic part tree 和 random noise vector 转化为由部分点云组成的三维形状点云 的映射，作为 symbolic part tree 的叶子节点
  2. 提出了 part-based conditional 点云 generator G() 和 discriminator D() 以symbolic part tree 输入为条件输入到
  3. 与整体点云生成整体形状点云不同，PT2PC 生成具有零件语义和形状结构的零件点云层次结构

3.1 Symbolic Part Tree Representation

  1. 如上图所示，遵循 PartNet 中定义的语义部件层次结构。每个 PartNet 形状实例(例如椅子)都用一个分层的部件分割进行注解，该分割既提供粗粒度级的部件(例如椅子底座、椅子靠背)，也提供细粒度级的部件(例如椅子腿、椅子靠背条)

  2. symbolic part tree 的定义为：

    • 表示一组 part instances（部件实例）
    • 中每个 part instance Pj 由两个组件组成，它们都用 one-hot vectors
    • semantic label Sj (e.g. chair seat, chair back)
    • part instance identifier dj (e.g. the first leg, the second leg)
    • 零件语义标签集是在PartNet中 pre-defined，并且在一个对象类别中是一致的
    • 中 每个边 (j, k) 表示 Pj是 PK的父节点

3.2 Part-tree Conditioned Generator

这一部分没看完

  1. conditional generator G() 取一个随机 Gaussian variable 还有一个 symbolic part tree condition 作为输入和输出的部分点云集合 xj形状空间中的部分点云是否代表叶节点部分 Pj

  2. 生成器由三个网络模块组成:

    • symbolic part tree encoder Genc
    • part tree feature decoder Gdec
    • part point cloud decoder Gpc

  3. 过程：

    • 首先，symbolic part tree encoder Genc 插入 的节点，对于每个节点 Pj，从叶节点到根节点，分层次地划分为紧凑的特征 tj
    • 然后，同时取随机变量 z 和符号部分特征 {tj}j 的层次结构
    • part tree feature decoder Gdec 以 top-down 的方式对部件特性进行分层解码
    • 接着，对每一个 Pj 节点生成 part feature fj
    • 最后，part point cloud decoder Gpc 在shape space 转换 叶节点 fj 进入 3D part point clouds xj

  4. 在零件特征解码的每一步，父节点都需要知道全局结构上下文，以便将相干信号传播到所有子节点，从而生成的零件点云能够以兼容的方式形成有效的形状

  5. 这就是为什么我们引入 symbolic part tree encoder Genc 作为一个 bottom-up 的模块来总结每个解码步骤的部分树结构上下文

  6. 然后，part tree feature decoder Gdec 以符号结构上下文为条件，以 top-down 的方式递归地将随机噪声 z 从根节点传播到叶节点

  7. symbolic part tree encoder Genc，对于给定的符号部件树 T ，我们从叶片节点开始对部件树的节点进行编码，并将消息传播到已编码节点的父节点，直到对根节点进行编码。消息传播以自底向上的方式执行

  8. 与现有作品整体生成形状点云相比，关键的区别在于，本文点云解码器分别为部分树 T 中的每个叶节点生成部分点云，但在某种程度上意识到部分之间的结构和关系

  9. 另一个很大的优势是，我们得到了每个生成的零件点云的语义标签。此外，我们观察到整体点云发生器的点分布通常不均匀。生成器倾向于分配更多的点给体积较大的部件(例如，椅背和座椅)，而只生成稀疏的点给几何形状较薄的小部件(例如，椅轮，椅背杆)

  10. 由于我们的Gpc为每个部分生成相同数量的点，然后进行全局下采样，所以我们可以为所有的部分生成结构精细、点密度合适的形状点云

3.3 Part-tree Conditioned Discriminator

  1. Discriminator 主要是判断每个部分点云的几何形状以及由这些部分组成的整体形状点云。这就是说，鉴别器应该告诉每个部分点云是否真实和可信的部分语义
  2. 此外，鉴别器需要观察零件点云的空间排列，判断它是否符合零件树所指定的真实结构
  3. 连接的部件需要相互接触，某些部件可能表现出某种对称性
  4. 最后，鉴别器应判断生成的零件点云形成逼真的形状点云
  5. 为了满足上述需求，鉴别器利用了两个模块：
    • 一个结构敏感的部件点云鉴别器
    • 一个整体形状点云鉴别器
