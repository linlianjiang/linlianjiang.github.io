21_ICCV_PointTr:Diverse Point Cloud Completion with Geometry-Aware Transformers

Created2022-11-10|Updated2022-11-10|paper learningPoint Cloud
paper code

0. Abstract

  1. PoinTr 采用 Transformer encoder-decoder architecture进行点云补全
  2. 点云是一组具有位置嵌入的无序点组，需要转换为点代理序列，并使用transformer生成点云
  3. devise a geometryaware block that models the local geometric relationships

1. INTRODUCTION

  1. 随着 PointNet 和 PointNet++ 的成功，直接处理 3D 坐标成为基于点云的 3D 分析的主流
  2. encoder-decoder的bottleneck 在于编码阶段的 max-pooling 操作，其中细粒度信息丢失并且在解码阶段难以恢复
  3. 将局部区域中的点云集表示为称为 Point Proxy 的特征向量
  4. Multi-Scale Point Cloud Generation：我们设计了一个多尺度点生成模块，以coarse-to-fine的方式恢复丢失的点云

2. Related

3D Shape Completion

  1. PCN 采用了folding net的框架，模仿2D平面变形将2D点映射到3D表面上（这部分得好好想想，要不要保留）

Transformer

  1. 3. Approach

3.1. Set-to-Set Translation with Transformers

  1. 主要目标是利用 Transformer 架构令人印象深刻的序列到序列生成能力来完成点云任务
  2. 符号解释
    • ME，MD：encoder，decoder
    • V=｛V1，V2 …VN｝: encoder输出feature
    • Q=｛Q1，Q2 …QM｝: decoder动态序列（dynamic queries）
    • H=｛H1，H2 …HM｝: 预测的缺失点云的点代理（Point proxy）
  3. encoder-decoder架构由 LE 和 LD multi-head self-attention 组成
  4. 编码器中的自注意力层首先使用 long-range 和 short-range information更新proxy features
  5. 然后 feed forward network (FFN) 用MLP更新proxy feature
  6. 解码器利用self-attention和cross-attention来学习结构知识
  7. 自注意力层利用全局信息增强局部特征，而交叉注意力层探索查询和编码器输出之间的关系
  8. 作者说要用Transformer代替传统的深度学习在点云上的应用。。。果然这是个趋势，这篇要好好看、

3.2. Point Proxy

  1. To make 3D point clouds suitable for transformers：
    • 第一步将点云转换为向量序列 ： 解决方案是直接将 xyz 坐标序列馈送到Transformer，但是这样消耗太多了
    • 所以要把原始的点变为一组 Point proxy，每组Point proxy代表一个区域的点云
    • 用最远点采样（FPS）在partial Point中间固定N个中心点，然后使用 light-weight DGCNN提取中心点特征

3.3. Geometry-aware Transformer Block

  1. 将 Transformer 应用于视觉任务的主要挑战之一是 Transformer 中的自注意力机制缺乏传统视觉模型（如 CNN 和点云网络）所固有的一些归纳偏差，这些模型明确地对视觉数据的结构进行建模
  2. 设计了一个 geometry-aware block，plug-and-play module ，包括 attention block 和 Transformer architecture
  3. 使用 kNN 模型来捕捉点云中的几何关系
  4. 给定查询坐标 pQ，根据键坐标 pk 查询最近键的特征。然后按照 DGCNN的做法，通过线性层的特征聚合和max pooing 操作来学习局部几何结构
  5. 然后将语义特征连接起来并映射到原始维度以形成输出

3.5. Multi-Scale Point Cloud Generation

  1. 目标是预测不完整点云的缺失部分，由于现在只能从Transformer decoder中获得缺失代理的预测， 因此，提出了一个多尺度点云生成框架，以全分辨率恢复丢失的点云
  2. 为了减少冗余计算，重用查询生成器生成的 M 坐标作为缺失点云的局部中心
  3. 然后，利用 FoldingNet 来恢复以预测代理为中心的详细局部形状（这个folding net每个点云补全都在用，一定要复现代码，看一下和原先的点云补全有什么不同）
  4. PF net是保持输入的形状，我们需要保持输入的形状，然后把生成的缺失点云和输入连接

4. Summary

  1. 首先现在的点云补全需要保持原先输入的形状，这点必须看PF net
  2. 将3D放到2D grid里，卷积的方法参考 folding net
  3. Transformer要成为趋势了，必须先把点序列化，首先用FPS，然后PointNet和DGCNN选一个，然后输入Transformer，适合多模态任务，一定要试一下
