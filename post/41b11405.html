20_ECCV_ReferIt3D_Neural Listeners for Fine-Grained 3D Object Identification in Real-World Scenes

Created2022-02-16|Updated2022-03-12|paper learningMultimodal
paper code

0. Abstract

Title: ReferIt3D: Neural Listeners for Fine-Grained 3D Object Identification in Real-World Scenes

  1. 首先开发了两个大规模和互补的视觉语言数据集:
    • Sr3D: 一个高质量的参考话语合成数据集。包含83.5K基于模板的话语，利用细粒度对象类之间的空间关系来定位场景中引用的对象
    • Nr3D：一个自然(人类)参考话语数据集。包含41.5K自然、自由形式的话语，通过在3D场景中部署一个2人对象引用游戏收集
  2. 利用这些数据，文章开发了novel neural listeners，可以理解以对象为中心的自然语言(object-centric natural language)，并在3D场景中直接识别被引用的对象
  3. 将语言和几何信息(以3D点云的形式)结合起来，并创建多模态(3D)neural listeners

1. Introduction

过去的作品大多专注于开发更好的模型，将视觉与图像中的语言联系起来，毕竟这只是我们的3D现实的2D视图，但Fine-grained 3D understanding也很重要

图1
  1. 每个用颜色编码对应的对象(用相同的颜色标记)与底层场景中分散注意力的对象进行区分;对比两把简单的椅子(左)和两把办公椅(右)。在场景中使用特定的对比上下文(由包围所有且仅包围相同细粒度类的对象的边界框所描绘)可以产生区分性语言，并将参考问题提升到细粒度分类之外。
  2. 对象类(object classes)的区分理解在细粒度级别非常重要，可以通过结合外观理解和空间推理技能的模型来实现
  3. 创建有区别的语言描述:专注于设计一个数据收集策略，涵盖基于空间(spatial)和外观(appearance)的识别
  4. 策略包括合成和基于人的话语生成，具有以下特点:
    • 在每个单独的例子中，都有同一个类的多个对象实例，在描述目标3D对象的语言中引用
    • 我们明确地要求人类主体描述目标对象，而不是同一对象类的其他实例。通过明确地对比相同的细粒度类实例，并且只对比它们，产生的话语是有区别的，即使是由不熟悉环境的众包注释者说出的
  5. Contributions:
    • 细粒度的ReferIt3D任务:我们引入了基于语言的特定3D对象实例识别任务，其中，细粒度的以对象为中心和多对象理解是其完成的必要条件。
    • Nr3D and Sr3D datasets
    • ReferIt3DNet: 通过设计一种新的视觉语言图神经网络(称为ReferIt3DNet 3)来探索理解基于真实世界3D数据(包括语言和场景)的对象引用的任务。

2. Related Work

3. Developing Referential 3D-Centric Data

  1. Sr3D和Nr3D都是建立在ScanNet之上，这是一个真实世界的3D场景数据集，具有广泛的语义注释，我们利用它来创建适当的对比通信上下文
  2. 将通信上下文定义为 (scene, target, distractor(s))
    • scene是ScanNet的707个独特室内场景之一
    • target是76个细粒度对象类之一
    • distractors是同一场景中包含的同一粒度对象类作为目标的实例
  3. 根据标准选择76个对象类，条件是：
    • 至少包含5个场景(scenes)
    • 每个场景包含多个干扰物，但不超过六个

3.1 Creating Template Based Spatial References

  1. 在Sr3D数据集中引入空间引用(Spatial Reference)
  2. 通过定义目标和周围对象(锚点)之间的关系，每个话语都旨在唯一地引用ScanNet 3D场景中的目标对象
  3. Anchors是可以属于ScanNet中100个对象类集合的对象实例(object instances)，但不能与目标属于同一类
  4. 例如，一个带有目标对象(如桌子)的潜在3D场景，可以通过与锚定对象(如门)的空间关系(如最接近的空间关系)从干扰物中完全消除歧义
  5. 我们使用以下合成模板合成有区别的Sr3D话语:
  6. “the $\color{red}{desk}$ that is $\color{blue}{closest}$ to the $\color{green}{door}$”：Sr3D模板由三个占位符组成。我们的目标是找到它们的组合，能够在场景中的干扰物中独特地描述目标物体
    图2
  7. 定义了以下五种spatial object-to-object relations:
    • Horizontal Proximity: 表示目标与场景中的Anchors有多近/多远。它适用于物体水平放置的距离
    • Vertical Proximity: 它表示目标在锚点之上或之下
    • Between: 表示两个锚之间存在一个目标
    • Allocentric: 编码关于目标位置与锚的内在自定向有关的信息
    • Support: 目标要么由锚支撑，要么支持锚

3.2 Natural Reference in 3D Scenes

  1. Nr3D数据集包含了通过在AMT中部署一个在线参考游戏收集到的41503个人类话语。这个游戏是在两个人之间进行的:speaker 被要求在ScanNet 3D场景中描述一个指定的目标物体，listener根据说话者的声音在干扰物中选择所指的物体

4. Developing 3D Neural Listeners

  1. 3D场景 S 表示为一个由 N 个点组成的 rgb 点云，单词标记化的话语 U = {u1，…ut} , 我们想要构建一个神经监听器来识别引用的目标对象

  2. 假设对 S = {O1，…， OM>} 表示在S中出现的对象(Oi)

  3. 将3D参考问题转化为一个分类问题，目标是在M个分割的3D实例中预测所指的“目标”

  4. 由于基本任务也是面向场景的，并且严重依赖于场景中呈现的对象的配置。由于这个原因，向神经侦听器提供包含其操作场景的明确信息的信号是很重要的

  5. 一个baseline是创建PointNet++层次场景特征(基于大量的 S)，我们将其与每个对象 Oi 独立提取的每个视觉表示融合在一起。虽然结果表示是同时以对象为中心和场景感知的，但它没有考虑到显式的对象对对象交互

  6. 一种更复杂的方法——它是 ReferIt3DNet 的一部分——使用结构化和显式的方式来捕获对象到对象的交互，以提供scene

  7. 具体来说，使用dynamic graph-convolutional network(DGCN)，该网络对场景中出现的对象的视觉特征进行操作(对象是图形的节点)。根据更新后的(每个节点)视觉特征之间的欧几里德相似度，在DGCN的每一层动态计算该图的边
  8. 如何将语言信号“融合”到一个像上面这样的管道中。尽管椅子在视觉上与门不同，我们的图形网络应该检查这些对象之间的关系，特别是当引用需要时(例如，“靠近门的椅子”)。为了促进这一行动，我们将视觉（对象）特征与语言特征（由 RNN 派生）融合，然后将它们传递给 DGCN
    图4
  9. A visual encoder(通过共享的PointNet++)处理给定场景中的每个3D对象，这些对象由包含其xyz坐标和RGB颜色的6D点云表示
  10. 描述被提及对象的话语(例如，“白板旁边的扶手椅”)由循环神经网络(RNN)处理
  11. 生成的视觉语言表示被融合在一起，并由动态图卷积网络(DGCN)处理，该网络为每个对象创建一个以对象为中心和场景(上下文)感知的表示
  12. DGCN的输出由MLP分类器处理，该分类器估计每个对象成为引用对象的可能性
  13. 在DGCN通过一个FG对象类分类器和一个参考文本分类器处理视觉语言表示之前，两个辅助损失会对其进行调制
