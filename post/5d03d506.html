19_IEEE_PartNet A Large-scale Benchmark for Fine-grained and Hierarchical Part-level 3D Object Understanding

Created2022-02-19|Updated2022-03-12|paper learningPoint Cloud
paper code

0. Abstract

Title: PartNet: A Large-scale Benchmark for Fine-grained and Hierarchical Part-level 3D Object Understanding

  1. PartNet：一个一致的、大规模的3D对象数据集
  2. 使用细粒度、实例级和层次化的3D零件信息进行注释
  3. 本文测试了用于细粒度语义分割的四种最先进的3D深度学习算法和用于分层语义分割的三种基线方法。我们还提出了一种新的零件实例分割方法，并展示了其优于现有方法的性能

1. Introduction

  1. PartNet数据集和三个基准测试任务
  2. 左图：展示了层次结构中三个分段级别的示例注释
  3. 右图：我们提出了三个基本且具有挑战性的分割任务，并使用PartNet建立了基准
  4. 现有数据集仅在相对较少的对象实例上提供零件注释，或在粗糙但非层次化的零件注释上提供零件注释，这限制了涉及理解细粒度和层次化形状分割的应用
  5. 现有的粗糙和同质零件理解算法无法在PartNet上很好地工作。首先，小而细粒度的零件（例如门把手和键盘按钮）非常丰富，对零件识别提出了新的挑战。第二，许多几何相似但语义不同的部分需要更多的全局形状上下文来区分。第三，理解形状和零件的异质性变化需要分层理解
  6. Contributions:
    • PartNet
    • 提出了三个部分级对象理解任务来证明这些数据的有用性：细粒度语义分割(fine-grained semantic segmentation)、层次语义分割( hierarchical semantic segmentation)和实例分割(instance segmentation)
    • 使用PartNet测试了四种最先进的语义分割算法和三种分层分割基线方法
    • 在PartNet上进行零件实例分割的任务，并描述了一种新的方法，其性能大大优于现有的baseline

2. Related work

  1. 来自ShapeNet的1016个3D模型，来自10个对象类别，具有手动注释的细粒度级零件语义，并研究学习神经网络，以便对细粒度零件组件进行分组和标记。PartNet提出了一个大规模数据集，其中包含573585个细粒度和分层形状零件注释，涵盖24个对象类别的26671个模型

3. Data Annotation

3.1 Expert-Defined Part Hierarchy

  1. 引入了And-Or-Graph-style风格的层次模板，并根据预定义模板收集了零件标注

  2. 由于要求设计的模板覆盖所有形状和零件的变化，总结了用来指导模板设计的标准:

    - Well-defined：**界限清楚**，部件概念被很好地描述，部件可以被多个注释器标识
    • Consistent：一致性，部件概念在不同的部件、形状和对象类别之间共享和重用
    • Compact：没有不必要的零件概念，零件概念在可能的情况下被重用
    • Hierarchical：分层次，部件概念组织在一个分类中，以涵盖粗粒度和细粒度的部件
    • Atomic：零件分类学中的叶节点由原始的、不可分解的形状组成
    • Complete：部件分类法尽可能完整地涵盖各种不同的形状

  3. 根据以上原则，为每个对象类别构建 And-Or-Graph-style 样式的部件模板

  4. 每个模板都以从粗语义部分到细粒度基元级组件的分层方式设计，如上图基于灯的模板

  5. And-nodes 将一个部件分割成小的子部件

  6. Or-nodes 将节点表示当前部分的子分类

  7. and -nodes和Or -nodes的组合使用相同的模板来覆盖结构上不同的形状，同时尽可能共享公共的部件标签

  8. 如上图(左)和(右)所示，台灯和吸顶灯都是通过相同的模板，通过灯类型的一级Or-node解释

  9. 但也不可能覆盖所有部分，所以允许注释器改进模板的结构，并注释超出定义范围的部分

  10. 为了解决零件的歧义问题，从每个类中保留一个形状子集，并为每个形状收集多个人工注释

3.2 Annotation Interface

  1. (a)显示了基于web的注释界面
  2. (b)基于模板层次结构，注释过程被设计成一个单线程的问题回答工作流，以深度优先的方式遍历模板图
  3. 从根节点开始，注解者被询问一系列的问题。答案自动为当前形状实例构造最终的层次分割。对于每个问题，注释器被要求标记子部分(And-node)的数量，或者在所有子类型(or -node)中为给定的部分选择一个。对于每个叶节点部件，注释器在3D界面中注释部件几何图形
  4. 注解者被询问一系列的问题。答案自动为当前形状实例构造最终的层次分割。对于每个问题，注释器被要求标记子部分(And-node)的数量，或者在所有子类型(or -node)中为给定的部分选择一个。对于每个叶节点部件，注释器在3D界面中注释部件几何图形。为了帮助他们理解部件定义和规范，我们为每个部件提供了丰富的文本定义和可视化示例。此外，我们的界面支持横截面和可见性控制来标注3D模型的内部结构
  5. 采集的3D CAD模型通常包含 original mesh subgroups 和 part information
  6. (c)在注释的开头为注释器提供了原始分组，以加快注释的速度。注释器可以简单地选择多个预定义的片段来组成最终分段的一部分。我们还提供网格切割工具，当原始分组比期望的分割更粗时，可以将大的部分分割成更小的部分

4. PartNet Dataset

  1. PartNet数据集为26,671个形状提供了细粒度的、分层的实例级的部件分割标注，其中包含来自24个对象类别的573,585个部件实例
  2. 大多数形状和对象类别都来自 ShapeNetCore
  3. 补充了室内场景中常见的3个对象类别(即剪刀、冰箱和门)，并在现有的7个类别中添加了来自 3D Warehouse模型

5. Tasks and Benchmarks

使用PartNet测试了三个部分级对象理解任务: 细粒度语义分割、层次语义分割和实例分割，性能优于baseline

5.1 Fine-grained Semantic Segmentation

细粒度三维语义分割需要识别和区分细小的和相似的语义部分。例如，门把手通常很小，在PartNet中平均为77分，但在门的语义上很重要。床有几个几何上相似的部分，如侧立杆、柱杆和基腿。为了识别细微的零件细节，分割系统需要通过判别特征局部地理解它们，并在整个形状的背景下全局地理解它们

Benchmark Algorithms

为了学习局部几何特征，用PointNet++提出了一种分层特征提取方案

Evaluation and Results

  1. 我们在每个对象类别的三个分割级别评估算法:粗粒度、中粒度和细粒度
  2. 在24个对象类别中，均为粗级，9个有中级，有17个中级。总的来说，我们为24个对象类别定义了50个分割级别

5.2 Hierarchical Semantic Segmentation

  1. 形状分割本质上是分层的。从粗语义到细粒度的细节，对 3D objects 的层次理解发展了对 形状组件的整体和全面的推理
  2. 层次语义分割问题，预测了覆盖粗粒度和细粒度零件概念的整个形状层次中的语义零件标签
  3. 层次分割的一个关键问题是如何在学习过程中利用给定形状模板上的丰富零件关系
  4. 层次分割需要对整个部分层次结构进行整体理解

Benchmark Algorithms

  1. 提出了三种 baseline methods 处理分层分割: bottom-up, top-down and ensemble（自底向上、自顶向下和集成）
    - bottom-up：在训练过程中只考虑叶节点部分，并将子节点的预测分组到自底向上推理的层次中定义的父节点 
- top-down：自上而下的方法学习树上所有零件语义标签上的多标签任务，并通过先分类较粗级别的节点，然后分类较细级别的节点来进行自上而下的推理
- ensemble： 在多个层次上训练平面分割，如第5.1，并通过计算树上所有根到叶路径的平均对数似然分数进行联合推断
- 使用PointNet++作为主干网络

Evaluation and Results

  1. bottom-up 只考虑训练中的叶节点部分，叶节点的父节点语义隐式编码在叶节点零件定义中。例如，椅子靠背和椅子扶手的竖条是两个不同的叶节点。top-down 的方法在培训和测试阶段都明确地利用了树结构。然而，预测误差是通过自上而下的推理累积起来的。集成方法将分层分割任务分解为多个级别的单个任务，并在考虑所有级别预测的情况下执行联合推理。虽然表现出更好的性能，但它有更多的超参数，并且对于多个网络需要更长的训练时间

5.3 Instance Segmentation

  1. 实例分割的目标是检测每个零件实例，并将其从形状上下文中分割出来

  2. 给定一个形状点云作为输入，零件实例分割的任务是在整个点云上提供几个不相交的遮罩，每个遮罩对应于对象上的单个零件实例

  3. 提出了一种新的分段检测网络来解决实例分段问题。网络架构上图所示。使用PointNet++作为主干网络，用于提取特征并预测每个点的语义分割和K实例分割掩码
