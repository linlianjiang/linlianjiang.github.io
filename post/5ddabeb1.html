16_CVPR_Natural Language Object Retrieval

Created2022-02-10|Updated2022-03-12|paper learningMultimodal
paper code

1. Introduction

  1. 通过领域自适应（ domain adaptation）和哈希算法（hashing），目标检测的类别数量已经从10K增长到100K

  2. 在预定义固定的对象类别集合（predefinedfixed set of object categories）中，人们喜欢用自然语言指代物体，而不是使用预定义的类别标签（predefined category label）

  3. 自然语言查询可以包括不同类型的短语，如：类别（categories）、属性（attributes）、空间配置（spatial configurations）（#空间结构分布）和与其他对象的交互（interactions with other objects）

  4. 如图，将自然语言对象检索框定为对给定图像中一组候选位置的检索任务；简单地将基于文本的图像检索系统应用于从该任务的候选位置裁剪出来的图像区域，会导致较低的性能；作为自然语言，对象检索涉及对象的空间配置和作为上下文的全局场景

  5. 例如：判断场景中的一个物体与“the man in a blue jacket sitting on the right in front of the house”的可能性有多大

    • 需要观察对象以确定它是否是 “the man”(category)，“in blue jacket” (attribute) 和 “sitting”(action)，以及它在场景中的空间配置以确定它是否 “on the right”
    • 整个图像作为全局上下文信息来确定它是否 “in front of the house”

  6. 图像检索技术：

    1. 基于文本的图像检索 (TBIR, Text Based Image Retrieval) ：利用文本标注的方式对图像中的内容进行描述，为每幅图像形成描述这幅图像内容的关键词，比如图像中的物体、场景等，这种方式可以是人工标注方式，也可以通过图像识别技术进行半自动标注
      • 文本检索过程实际上可以理解为文本特征匹配的过程，以上过程文本使用词袋向量（Bag-of-Words，BoW）来表征文本内容
      • BoW特征提取过程包括以下几个步骤：
        • 将文档中的文本解析成单词 ；
        • 使用词干表示各单词,去掉常用词；
        • 统计当前文档中每个单词（词干）出现的频次；
        • 建立表示该文档的向量，向量的每次元素代表对应位置的单词出现的频次，向量长度等于词库内单词个数。在构建词向量时，通常会引入加权机制。常用的一种加权方式为采用 tf-idf (term frequecy-inverse document frequency) 。 tf-idf的计算方式如下：

    2. 基于内容的图像检索(CBIR, Content Based Image Retrieval) ：使用视觉特征来表示， 对图像提取若干个局部特征描述子，如sift，对这些描述子进行量化。量化器通常通过聚类得到：对特征描述子集合进行k-means聚类，聚类后得到的k个质心即为视觉单词。描述子desc的量化结果q(desc)为与desc最相近的质心的索引。所有质心构成了视觉词表。图像中的特征单词的词频构成了该图像的向量描述BoVW。假设视觉词表中的单词个数为N，那么BoVW向量的长度为N，向量中的元素为对应单词出现在该图像中的频次或者采用采用td-idf权重更新向量中每个元素值。

    3. 自然语言图像检索方法：在给定输入图像、文本查询和一组候选位置(例如对象建议方法)的情况下，使用循环神经网络模型（recurrent neural network）对基于局部描述符、空间配置和全局上下文的候选位置进行评分。检索得分最高的候选人。

  7. 本文提出空间上下文循环卷积网络(Spatial Context Recurrent ConvNet，SCRC)模型来学习一个评分函数，该函数将文本查询、候选区域、它们的空间配置和全局上下文作为候选区域的输入和输出评分

  8. 使用了一个两层的LSTM（Long short-term memory）网络结构，嵌入的文本序列和视觉特征分别作为第一层和第二层的输入

  9. 相比其他类型的 visual-linguistic 模型如 bag-of-words ,使用 recurrent neural network 作为scoring function的优点之一是简单的反向传播整个模型很容易地学会了端到端通过,允许视觉特征提取和文本序列嵌入互相适应,它的性能明显优于之前使用单词袋的方法

  10. 自然语言对象检索的主要挑战之一是缺乏带有标注对象边界框和描述对的大规模数据集

    • 为了解决这一问题，我们展示了通过先在图像标题领域进行预处理，然后将其调整到自然语言对象检索领域，可以将从前一个任务中学习到的视觉语言知识转移到后一个任务中
    • 这种预训练和自适应过程提高了性能，避免了过拟合，特别是在目标检索训练数据集较小的情况下

2. Related work

  1. 基于来自ImageNET分类器的单词、句子模型和嵌入包，解决了类似的问题，并基于文本查询在图像中定位对象
  2. 给定一组候选对象区域，使用大规模预训练分类器预测的类别名称，从这些表示为词袋的候选对象生成文本，并将词袋与查询文本进行比较
  3. 其他方法从查询文本生成视觉特征并将其与图像区域匹配，例如通过基于文本的图像搜索引擎或学习文本短语和视觉特征的联合嵌入
  4. 通过recurrent network model来根据给定的描述定位对象

3. Model

用自然语言对象检索的空间上下文循环卷积网络(SCRC)模型和训练过程。

在测试时，提供了一幅图像、一个自然语言对象查询和一组候选边界框(例如来自对象建议方法，如EdgeBox)。系统需要从候选集合中选择匹配查询文本的边界框子集。

3.2 Knowledge transfer from image captioning

  1. 经过预训练，我们将SCRC模型应用于自然语言对象检索
  2. 假设训练数据集由 N 幅图像组成，每幅图像包含 Mi (i = 1,…N)个标注对象,每个对象都被一个边界框（bounding box）标注，Ki,j文本描述(i= 1,…, N , j = 1,…, Mi)。一个对象可以用不同的描述进行多次描述
  3. 在训练时，每个实例都是一个图像包围框描述元组（image-bounding box-description tuple）(Ii, bi,j, Si,j,k)
    • Ii 是整个图像
    • bi,j = [xmin, ymin, xmax, ymax]是第 j 个物体的边界框(bounding box)
    • Si,j,k 是自然语言的描述文本，such as“the black and white cat”

  4. 本文的自然语言对象检索模型可以通过以局部图像区域 Ibox 和整个图像 Iim 为上下文的ground truth注释中对象描述文本的概率最大化来训练，类似于训练一个通用的目标检测系统

  5. 自然语言对象检索中，对象的描述文本(description text)可以看作是对象的一个广义“标签”，最大化其条件概率类似于训练一个“广义分类器”，其输出是一个单词标签序列，而不是单个类别标签

  6. 给定一个自然语言对象检索数据集，我们从ground truth注释中构造所有元组(Ii, bi,j, Si,j,k) 作为训练实例(如果对同一对象有多个描述，则构造多个元组)

  7. 对于训练集中的每个标注对象，基于该目标区域的边界框(bounding box)从整个图像 Iim 中裁剪出一个image patch Ibox ，其空间构型 Xspatial 通过方程构造

4. Experiments

  1. 在ImageNET上训练一个7K object classes的large scalefine-grained classifier
  2. 候选集合中的每个盒子都被划分到7K class中的一个,根据包含类别名称和同义词的ImageNET 同义词集合，从预测对象类中提取单词包
  3. 然后，将单词包投影到向量空间，利用余弦距离与投影的查询文本进行匹配，得到 score

4.1 Object retrieval evaluation on ReferIt dataset

  1. ReferIt 数据集：

    • 包含图像区域的最大的公开可用数据集

    • 它包含了来自IAPR TC-12数据集的20,000张图像

    • 结合SAIAPR-12数据集的分割图像区域

    • 以及针对旨在从注释中识别图像区域的双人游戏中收集的图像区域的120K注释描述

    • 也包含一些模糊的(例如“anywhere”)和错误注释的例子，注释不对应任何对象

  2. 为了对该数据集进行评估，将2万张图像(以及它们的注释)分割为1万张用于训练和验证，1万张用于测试，并在所有带注释的图像区域上构造图像包围框描述元组作为训练实例

  3. trainval集中有59976个元组(image, bounding box, description)，测试集中有60105个元组

  4. 数据集的实验中，训练和评估时只使用标注区域的边界框，边界框由标注器点击对应的SAIAPR-12数据集中的分割区域获得

  5. 为了测试结合空间配置 Xspatial 和scene-level 上下文特征 xcontext 的效果，在ReferIt的微调中评估不同的设置

    • 通过在微调和测试期间将 Xspatial 和 Wglobal 设置为0
    • 该模型只能学习根据 candidate boxes中的局部图像描述符 Xbox为盒子打分
    • 类似地，通过将 Wglobal 设置为0，模型可以学习 Xbox和 Xspatial 上的评分函数，但不能利用 scene-level 上下文

  6. 作为对比，我们直接在ReferIt上训练SCRC模型，而没有对MSCOCO进行预先训练，在训练和测试时，将 Xspatial 和 Wglobal 设置为0，denoted by SCRC (w/o context, spatial, transfer)

  7. 模型中的CNN参数由VGG-16 net初始化，其他参数随机初始化

  8. 我们使用两个测试场景进行评估:

    检索到的区域与ground truth边界框至少有50%的IoU重叠，则被认为是正确的

    • 在第一个场景中，给定一个图像和一个文本查询，该模型被要求从该图像中所有带注释的区域中检索相应的图像区域
    • 在第二个场景中，给定文本查询，模型从对象建议方法产生的一组候选边界框中检索图像区域（这是一个更难的任务，但更接近真实的应用程序）

4.2 Object retrieval evaluation on Kitchen dataset

  1. Kitchen数据集：

    • 包含从ImageNET层次结构的Kitchen /household子树中采样的606张图像，每个图像有10个不同的描述注释

    • 由于该数据集中的对象几乎占据了整个图像，在候选对象建议框中不使用检索

  2. 在测试过程中，对于每个查询文本，候选集由11个具有ground truth和10个干扰物的图像组成。干扰物的采样要么来自同一个Kitchen数据集(“Kitchen”实验)，要么来自整个ImageNET(“ImageNET”实验)
  3. 为了在这个数据集上评估，将数据集分成两部分：以300幅图像作为trainval集，306幅图像作为测试集
  4. 与4.1节类似，首先在MSCOCO数据集上预训练SCRC模型用于image captioning，然后在trainval集合上对模型进行微调
  5. 型通过对ground truth和10个干扰物的候选集合进行 image-level 级检索来测试，使用从整个图像中提取的特征作为 Xbox
  6. 由于数据集不涉及 spatial configurations 或 scene-level 的上下文信息，在微调和测试期间，我们将 Xspatial 和 Wglobal 设置为0

5. Conclusion

  1. 本文提出了Spatial Context Recurrent ConvNet(SCRC)的自然语言对象检索方法。SCRC是一种基于局部图像描述符、空间配置和全局场景级上下文对候选框进行评分的recurrent neural network model。我们发现，将空间结构和全局上下文结合起来，可以显著提高自然语言对象检索的性能。在我们的方法中使用的循环网络模型导致了端到端可训练的评分函数

  2. 此外，我们还证明了自然语言对象检索可以通过预训练(pretraining)和自适应(adaptation)来传递在 image captioning 上学习到的知识

  3. 由于自然语言对象检索系统的困难之一是缺乏具有 object-level 注释的大型数据集，但利用带有image-level注释的数据集可以缓解这个问题，image-level 注释通常比 object-level 描述更容易收集
