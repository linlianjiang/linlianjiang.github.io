22_ECCV_Masked Discrimination for Self-Supervised Learning on Point Clouds

Created2022-08-16|Updated2022-11-07|paper learningPoint Cloud
paper code

0. Abstract

Title: Masked Discrimination for Self-Supervised Learning on Point Clouds

  1. MAE（mask autoencoder）在图像和语言领域的自监督学习中取得了巨大的成功
  2. 然而，基于掩码的预训练还没有显示出对点云理解的好处，这可能是由于像PointNet这样的标准主干无法正确处理在训练期间掩码引入的训练与测试分发不匹配
  3. 本文通过提出一种用于点云的鉴别掩模预训练Transformer框架MaskPoint来弥补这一差距
  4. 关键思想是将点云表示为离散的占用值(1表示点云的一部分;如果不是，则为0)，并对掩蔽对象点和采样噪声点进行简单的二进制分类作为代理任务
  5. 在多个下游任务中评估我们的预训练模型，包括3D形状分类、分割和真实物体检测，并展示了最先进的结果，同时实现了与之前最先进的Transformer基线相比的显著的预训练速度(例如，在ScanNet上的4.1×)

1. Introduction

  1. MAE：预测句子中的 mask word或 image中的 mask patch
  2. 点云获得高质量的注释既困难又昂贵，特别是对于真实世界的扫描。同时，掩码自动编码也应该很适合点云数据，因为每个点(或一组点)都可以很容易地掩码或解掩码
  3. main idea: 将点云随机划分为掩码集和非掩码集。我们只将点云的可见部分输入编码器。然后，从掩蔽点中抽取一组真实查询点，从三维空间中随机抽取一组虚假查询点。我们训练解码器，使其区分真点和假点。经过预训练后，我们丢弃解码器，使用编码器进行下游任务
  4. 我们假设，mask autoencoder 迄今为止对点云数据效果不佳的主要原因是，标准的点云主干无法正确处理掩码引入的训练数据和测试数据之间的分布不匹配。具体来说，PointNet类型骨干网利用了在每个点的局部邻居(例如，k近邻)上操作的局部聚合层。随着掩蔽的引入，当地社区的范围可能会发生巨大变化，在掩蔽训练场景和未掩蔽测试场景中看到的当地社区分布之间产生差异。另一方面，transformer可以对输入数据的全部或部分执行self attention(一种聚合形式)。这意味着它有能力在训练数据中只处理场景的 unmask 部分，而不受mask部分的影响。这个属性表明，transformer可以成为点云的 Self-Supervised 屏蔽 autoencoder 的理想主干选择
  5. 为了理解图像，最先进的掩码自编码Transformer方法MAE掩码了大量随机的图像补丁子集，将Transformer编码器应用于未掩码的补丁，并训练一个小型Transformer解码器，该解码器接收掩码补丁的位置编码，以重建其原始像素值
  6. 然而，这种方法不能直接应用于点云数据，因为每个3D点的原始表示是其空间xyz位置。因此，训练解码器预测掩蔽点的xyz坐标是很简单的，因为它的位置编码将泄露正确的答案。在这种情况下，网络只会走捷径，而不会学到有意义的特征。
  7. 为了解决这个问题，我们提出了一个简单的二进制点分类目标作为一个新的借口任务的点云掩码自动编码。我们首先将点分组到当地社区，然后屏蔽掉这些群体中的一个大的随机子集
  8. Transformer编码器接收 unmask 的点组，并通过对其他组的 self-attention 对每个组进行编码。Transformer解码器接收一组真实和虚假的查询点，其中对真实查询进行抽样 而伪查询则是从整个3D空间中随机采样的。然后，我们在解码器查询和编码器输出之间执行交叉注意。最后，我们对解码器的输出应用一个二进制分类头，要求它区分真实和虚假查询。我们发现这种简单的设计是有效的，需要网络从少量可见的点组推断出物体的3D形状
  9. a standard Transformer

2. Related Work

  1. Pretraining为基础的 mask。屏蔽内容已被用于各种方法来提高模型的鲁棒性，包括正则化[47,17]、数据增强[13,46,74]和自监督学习[7,12,20]
  2. 对于自监督学习，其关键思想是训练模型根据其周围环境预测被屏蔽内容。最成功的方法建立在Transformer之上，部分原因是它基于标记的表示和建模远程依赖关系的能力
  3. MAE 在像素级上重构 mask patch，而不是重构 token，其 mask rate 更高，≥70%
  4. 接下来的工作试图预测更高级的视觉特征，如HoG[58]，或者通过将可见补丁和掩码补丁[7]的特征对齐来提高编码器的表示能力
  5. 据我们所知，point - bert是唯一一种用于点云的自监督掩码建模Transformer方法，它采用了与BEiT[2]类似的思想
  6. 一些并发工作[36]也探索了在点云上采用掩码自编码预训练目标。与这些当代方法相比，我们的关键新颖性是判别性点分类目标，它有助于解决点云中的采样方差问题
  7. Mask language modeling: BERT, mask token， context sensitive
  8. 点云上 用Transformer做 MAE：Point BERT（22 ），这篇22 ECCV，

3. Approach

  1. 即使是人类，也很难或不可能精确地重建所有缺失的点，因为有几个模糊的因素;例如，机翼的精确厚度或飞机的精确长度。如果我们在空间中得到一个采样的3D点，并被要求回答它是否可能属于该物体，我们会对自己的答案更有信心。
  2. 这种区分性的点分类任务比重建任务的模糊性要小得多，但仍然需要对对象语义的深入理解，以便从少量可见点中推断出被掩盖的点
  3. 重建 转化为 分类任务

3.1 Masked Point Discrimination

  1. 将每个输入点云P∈RN×3随机划分为两组:掩码M和未掩码U
  2. 使用Transformer编码器通过 self-attention 对稀疏分布的 unmasked tolen U之间的相关性建模
  3. 理想情况下，得到的编码 latent representation tokens L 不仅要对 unmasked U之间的关系建模，还要恢复masked points M的潜在分布，这样才能很好地完成预训练任务
  4. 我们采样一组真实查询点Qreal和一组虚假查询点Qfake。真实查询点从掩蔽点集M中采样，虚假查询点从整个三维空间中随机采样。然后我们在每个解码器查询q∈{Qreal, Qf ake}和编码器输出CA(q, L)之间进行交叉注意，以建模掩码查询点和未掩码点之间的关系
  5. 最后，我们对解码器的输出应用一个二进制分类头，要求它区分真实和虚假查询

Discarding Ambiguous Points

  1. 由于我们在整个空间中均匀随机采样伪查询点，因此会有一些点落在接近对象表面的地方。这些点由于其目标标签，会造成训练困难
  2. 用PointNet将 unmask (Visible Point Groups) 的点到 token embeddings
  3. 然后，从 masked point 中抽取一组真实查询点，从三维空间中随机抽取一组虚假查询点
  4. 训练解码器，使其区分真点和假点
  5. 经过预训练后，我们丢弃解码器，使用编码器进行下游任务

3D Point Patchification

  1. 将输入的点云转换为三维点补丁
  2. 给定输入点云P∈RN×3，使用最远点采样对S个点{pi}Si=1进行采样作为补丁中心
  3. 然后收集每个patch中心的k个最近的邻居，生成一组3D点patch {gi}Si=1
  4. 然后使用PointNet将每个3D点补丁gi∈Rk×3编码为一个特征嵌入fi∈Rd，得到S个令标及其对应的特征{fi}Si=1，中心坐标{pi}Si=1

3.2 Why not reconstruction, as in MAE?

  1. 在本节中，我们深入研究了为什么在相关的蒙面自动编码器(MAE)方法中用于图像的重建目标(即从mask的点重建原始点云)不能用于我们的点云设置的细节
  2. 首先，在MAE中，自我监督学习任务是根据输入图像的未屏蔽(可见)补丁重建被屏蔽的补丁
  3. 具体来说，给定每个掩码图像patch查询的2D空间位置，目标是生成其RGB像素值
  4. 在我们的例子中，类似于为一个被屏蔽的3D点补丁查询生成空间xyz值—这对于模型来说是微不足道的，因为查询已经包含了相应的空间信息。这样一个微不足道的解决方案将导致完美的零损失，并阻止模型学习有意义的特征表示

4. Experiments

  1. 90% mask ratio, 重构性好
  2. 可能会有多个可信的重建
  3. 更仔细地观察入住率分布。虽然被掩盖点云的红色区域和紫色区域都不存在点，但重构后的概率分布正确地反映了原始点云的概率分布:红色区域占比较低，紫色区域占比较高
