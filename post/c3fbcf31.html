21_arxiv_learning to prompt for vision-language models

Created2022-03-30|Updated2022-03-30|paper learningMultimodal
paper code

0. Abstract

Title: learning to prompt for vision-language models

  1. CoOp ：context optimization
  2. 使用 continuous representations 和 从数据中执行端到端的学习在保持预训练参数不变
  3. 结果：只需一到两张照片，就能在相当大的范围内击败手工制作的提示，当使用更多照片时，能够获得显著的改进

1. INTRODUCTION

  1. 视觉表示学习的传统方法是训练视觉模型来预测使用离散标签的一组固定的对象类别

  2. 这种方法将视觉识别系统限制在训练期间定义的封闭集合的视觉概念，使得它们一旦部署在目标环境中，就无法处理新的类别，因为学习新的分类器需要额外的数据

  3. 对于预先训练的视觉语言模型，文本输入，即 prompt（提示符），在下游数据集中扮演着关键的角色

  4. 识别正确的 prompt 是一项艰巨的任务，它通常需要大量的时间进行文字调优——因为措辞上的微小变化可能会对性能产生巨大的影响

  5. 例如上图 a 中

    • 2nd vs 3nd , 在 class token 前添加 “a” 可以带来超过 5% 的准确性提高
    • b - d 添加与任务相关的上下文可以导致显著的改进 i.e. ( “flower” for Flowers102, “texture” for DTD and “satellite” for EuroSAT )
    • 调整句子结构可以带来进一步的改进: “a type of flower” 放在Flowers102的类令牌之后，在DTD的上下文中只保留“纹理”，并在EuroSAT的“卫星照片”之前添加“居中”。然而，即使进行了广泛的调优，所得到的提示也不能保证对这些下游任务是最优的

  6. 本文提出 context optimization （上下文优化）(CoOp) 来自动化提示工程，以允许更有效和任务特定的转移预训练的视觉语言模型

  7. 如上图所示，本文使用 continuous representations 对 prompt 的 context （上下文）进行建模，这些表示本质上是用与单词嵌入具有相同维数的随机向量初始化的

  8. context可以在所有类之间共享，也可以被设计为 class-specific

  9. 在训练过程中，我们简单地利用相对于可学习上下文向量的交叉熵损失最小化预测误差，同时保持预先训练的参数不变

  10. 梯度可以通过文本编码器反向传播，提取出与学习任务相关的上下文参数中编码的丰富知识

2. METHODOLOGY

2.1 Vision Language Pre-training

Models

  1. CLIP 由 image encoder 和 text encoder 组成
  2. given a sequence of words (tokens)，such as “a photo of a dog.”
    • CLIP首先将每个标记(包括标点符号)转换为小写的 byte pair encoding （字节对编码）(BPE)表示，它本质上是一个唯一的数字ID
    • 为了方便 minibatch processing（小批处理），每个文本序列都包含了 [SOS] 和 [EOS] 标记，并以77为固定长度
    • 之后，这些 ID 被映射为512-D的 word embedding vectors （单词嵌入向量），然后传递给Transformer
    • 最后，对 [EOS] token position 的特征进行层归一化处理，并由线性投影层进一步处理

Training

  1. 给定一批 image-text pairs，CLIP使匹配对的余弦相似度最大化，同时使所有其他不匹配对的余弦相似度最小化
  2. 为了学习更多可转移到下游任务的不同视觉概念，CLIP的团队收集了一个由4亿个图像-文本对组成的大型训练数据集

3. EXPERIMENTS

3.1 FEW-SHOT LEARNING

Datasets

  • ImageNet
  • Caltech101
  • OxfordPets
  • StanfordCars
  • Flowers102
  • Food101
  • FGVCAircraft
  • EuroSAT
  • UCF101

Training Details

  1. CoOp有四个版本:
    • 将 class token 定位在末尾或中间
    • 统一上下文vs. CSC（class-specific context）
  2. 从11个数据集的实验中可以看出，CoOp均超过了CLIP，并且在一些数据集上，大幅度超过CLIP。证明了可学习的prompt优于人为设计的prompt。并且CoOp提出的两种变体，在一些数据集中效果更好
  3. 除非另有说明，ResNet-50 被用作 image encoder 的主干，context token 的数量 $M$ 被设置为16

Baseline Methods

  1. 本文将CoOp与两种 baseline methods 进行比较

    • 第一个是 zero-shot CLIP，它基于手工制作的提示

      • 对于一般的对象和场景，“ a photo of a [CLASS] “

      • 对于 fine-grained categories，添加了任务相关的上下文，比如为OxfordPets添加了“ a type of pet”，为 Food101添加了 “a type of food”

      • 当涉及到诸如识别DTD中的纹理之类的专门化任务时，提示符被定制为“[CLASS] texture”。类名是形容词，如“bubbly” and “dotted

    • 第二个 baseline 是 linear probe （线性探测）CLIP，它基于手工制作的提示

      • 在高质量的预训练模型特征(如CLIP)之上训练线性分类器，可以轻松实现与最先进的少拍学习方法同等的性能
  • 本文采用与Radford et al.(2021)相同的训练方法来训练 linear probe CLIP

3.2 ROBUSTNESS TO DISTRIBUTION SHIFT

  1. 由于CoOp需要对特定的数据分布进行训练，它有学习虚假相关性的风险，而这些虚假相关性不利于在不可见的分布(领域)中的泛化，这是最近的研究中提出的
  2. 相反，zero - shot CLIP不依赖于特定的数据分布，对分布转移具有很强的鲁棒性
  3. this section, 目标是揭示CoOp对分布转移的鲁棒性，与 zero - shot CLIP 和 linear probe CLIP 模型相比

3.3 FURTHER ANALYSIS

Comparison with Prompt Ensembling

  1. Radford 建议通过对使用不同 hand-crafted (手工) 提示生成的多个 zero-shot classifiers 进行集成来获得额外的改进，such as “a photo of the large [CLASS].”, “a bad photo of the [CLASS].” and “a origami [CLASS].” 为一幅图像分别反映不同的尺度、视图和抽象
  2. 我们很想知道，与 Prompt Ensembling 相比，CoOp学习的 prompt 是否仍然具有优势
  3. 结果：Prompt Ensembling 效果好，CoOp还有改进空间

5. CONCLUSION

6. 自己的总结

  1. 作者设计了一个可学习的模板来替代固定的模板，添加词或者改变prompt的位置，效果差别很大
  2. CSC, class-specific context
  3. prompt的长度长点，信息量可以学的更充分
  4. 本文作者提出了一种context optimization（CoOP）的方法，通过构造soft prompt方式，即prompt是参数化，结合离散标签使用continuous representation建模上下文，并在保持预训练学习的参数固定的同时从数据中进行端到端学习优化，让网络学习更好的prompt
  5. 这样，与任务相关的prompt设计就可以完全自动化了
  6. 实验结果表明，CoOP在11个数据集上有效地将预训练的视觉语言模型转化为数据高效的视觉任务学习模型，只需少量样本微调就能击败手工设计的提示符，并且在使用更多样本微调时能够获得显著的性能提升
