Created2022-02-24|Updated2022-03-12|paper learningPoint Cloud

0. Abstract

Title: ShapeNet: An Information-Rich 3D Model Repository

  1. ShapeNet：一个有丰富注释的大型形状存储库，由 object 的3D CAD模型表示
  2. ShapeNet包含 multitude of semantic categories的3D模型，并根据WordNet分类法对其进行组织
  3. 它是一组数据集，为每个3D模型提供许多 semantic annotations，例如 consistent rigid alignments, parts and bilateral symmetry planes(双侧对称平面 ), physical sizes, keywords, 以及other planned annotations
  4. 注释通过基于web的公共界面提供，以实现对象属性的数据可视化，促进数据驱动的几何分析，并为计算机图形学和视觉研究提供大规模量化基准

1. Introduction

  1. 计算三维形状的分割并建立它们之间的对应关系是几何形状分析中的两个基本问题
  2. 所有研究问题的根源: 将语义附加到三维形状的表示中
  3. Contributions:
    • Collect and centralize 3D model datasets, helping to organize effort in the research community.
    • Support data-driven methods requiring 3D model data
    • 能够评估和比较涉及几何的基本任务: segmentation, alignment, correspondence

2. Background and Related Work

  1. 大多数以前的收集都是为了评估形状检索和分类算法而开发，包含了很少的结构和语义注释

3. ShapeNet: An Information-Rich 3D Model Repository

  1. ShapeNet 包含跨越多种语义类别的模型
  2. 为每个模型提供了大量的注释集，以及存储库中的模型与存储库外的其他多媒体数据之间的链接
  3. ShapeNet根据WordNet语法集以分层分类的方式提供了所包含数据的视图
  4. ShapeNet的注释包括 geometric attributes，例如 upright and front orientation vectors, parts and keypoints, shape symmetries（reflection plane, other rotational symmetries）以及以真实世界单位表示的对象比例
  5. 根据WordNet分类法对其中的30万个进行了分类

Language-related Annotations

  1. 对相关数据源的 basic category 命名： index, group, link
  2. Synsets(句法集)与各种关系: hyper and hyponym, and part-whole relations(上下位词和部分-整体关系 )相互关联
  3. 有关基本类别注释的获取和验证的详细信息，请参考第4.1节

Geometric Annotations

  1. 算法预测和手动注释相结合，根据类别级几何特性组织形状，并从原始三维模型几何体中进一步导出丰富的几何注释
  2. Rigid Alignments: 为每个模型建立一致的规范方向（例如，直立和正面）能够使用分层聚类和对齐方法，以确保每个类别内一致的刚性对齐（见第4.2节）
  3. Parts and Keypoints: 几何结构和形状相关的重要关键点和语义信息，例如，不同的材料通常与不同的零件相关联
  4. Symmetry：（对称性）双侧对称平面和旋转对称在人工和自然物体中普遍存在，并且与形状的对齐和功能有着密切的联系。
  5. Object Size ：对象大小在许多应用中都很有用，例如在对象识别中减少假设空间。第5.2节讨论了尺寸标注

Functional Annotations

  1. Functional Parts: 根据零件的功能对其进行注释——事实上，零件的定义必须基于几何和功能标准
  2. Affordances: (能供性) 对功能和活动特定的启示注释。此类注释的示例包括支持平面注释和用于各种对象操作的可抓取区域注释（ supporting plane annotations, and graspable region annotations for various object manipulations）

Physical Annotations

  1. Surface Material: optical properties and semantic names of surface materials

3.3 Annotation Methodology

  1. 首先通过算法预测每个模型实例的注释。例如， global symmetry planes, consistent rigid alignments（全局对称平面、一致刚性对齐）
  2. 然后，我们将通过众包管道和人类专家的检查来验证这些预测

4. Annotation Acquisition and Validation

构建ShapeNet的一个关键挑战是获取和验证注释的方法

4.1 Category Annotation

如第3.2节所述，我们将每个3D模型分配给WordNet分类法中的一个或多个语法集

Annotation

  1. 模型通过文本查询检索到收集的在线存储库，将初始类别注释设置为每个检索到的模型的已用文本查询
  2. 检索这些模型后，我们使用存储库中每个模型的 popularity score（流行度分数）对模型进行排序，并要求人工验证指定的类别注释
  3. 这是合理的，因为更流行的模型往往是高质量的，并且通过category关键字文本查询正确地检索到

Clean-up

  1. Single 3D models： semantically distinct objects ( 语义不同的对象 ) focus: ShapeNetCore

  2. 3D scenes: 通过计算体素化表示中连接组件的数量来检测。手动验证这些检测并标记场景以供将来分析

  3. Billboards: 具有绘画纹理的平面。通常用来代表人和树。这些模型通常不适用于几何分析。可以通过检查单个平面是否适合所有顶点来检测它们

  4. Big ground plane:

    将单个3D models 包含在ShapeNet的ShapeNetCore 中

4.2 Hierarchical Rigid Alignment

为每个类别中的模型建立一致的规范化方向

这种对齐对于各种任务都很重要，例如 visualizing shapes, shape classification and shape recognition

上图显示了ShapeNet中一致对齐的几个类别

  1. “armchai”、“chair” 和 “seat” 是我们分类法中的三个类别，每个类别都是其后续类别的子类别
  2. 对于“armchai”类别中的形状，可以很好地定义一致的方向，通过检查 arms , legs , backs ,然而，很难定义“chair”的类别
  3. 例如，“ side chair ” 和 “ swivel chair”都是“ chair ”的子类别，然而，swivel chair的腿结构与side chair不同
  4. 对“ seat ”的定义变得更加模糊，它有“ stool ”, “couch”, and“chair”等子类别
  5. 提出一种分层对齐方法，按照ShapeNet的分类法，以自下而上的方式对模型进行分层对齐
  6. 例如，从排列 low-level categories中的形状开始，然后逐渐提升到更高级别类别，进入更高的层次时，应该保持子类别中的自我一致性方向

4.4 Symmetry Estimation

对称估计

  1. 在 ShapeNetCore 中为所有的3D模型提供了 bilateral symmetry plane detections （双边对称平面检测）
  2. 基本思想：利用霍夫变换对对称平面的参数进行投票，从网格中生成所有顶点对的组合。在均匀划分的平面参数离散空间中，每对对一个可能的对称平面投一票，然后选取得票最多的参数作为对称平面的候选参数。作为最后一步，验证这个候选点，以确保每个顶点都有一个对称的对等点

4.5 Physical Property Estimation

  1. 在计算物理属性注释之前，模型的维度需要与现实世界相对应
  2. 在给定的绝对尺寸下，我们现在通过填充体素化计算每个模型的整体体积
  3. 使用Binvox实现的空间雕刻方法，已知的类似容器的对象类别(例如，瓶子，微波)被标注为这样，并且只使用表面体素化体积
  4. 然后，估计每个对象类别的比例材料组成，并使用材料密度表和每个模型实例体积来计算该实例的粗略总重量估计
