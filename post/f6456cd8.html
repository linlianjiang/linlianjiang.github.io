21_ICCV_Towers of Babel_Combining Images, Language, and 3D Geometry for Learning Multimodal Vision

Created2022-03-02|Updated2022-03-12|paper learningMultimodal
paper code

0. Abstract

Title: Towers of Babel: Combining Images, Language, and 3D Geometry for Learning Multimodal Vision

  1. 3D-augmented collections的主要信息来源——language，例如图像说明(image captions)还未开发
  2. 本文提出 WikiScenes，新的大规模数据集，包含了以标题(captions)和层次类别名称形式(hierarchical category names)出现的描述性文本(descriptive text)
  3. WikiScenes为涉及图像、文本和3D几何图形的多模态推理形成了一个新的测试平台
  4. 演示WikiScenes在图像和3D模型上学习语义概念(semantic concepts)的效用。我们的弱监督框架连接图像、3D结构和语义——利用3D几何提供的强约束，将语义概念与图像像素和3D点联系起来

1. Introduction

  1. 自然语言是描述3D世界复杂性的有效方法;3D场景展示了诸如组合性(compositionality)、物理和功能关系(physical and functional relationships)等特征，这些特征很容易被语言捕捉到
  2. WikiScenes数据集：包含63K对图像和描述，捕获了99座教堂，以及它们相关的3D重建
  3. 关键观点：原始文本描述代表了对语义概念的一种微弱的、有噪声的监督形式，但场景的底层3D结构产生了强大的物理约束，可以增强数据噪声的鲁棒性，并可以ground models
  4. key contributions：
    - WikiScenes数据集，结合了语言、图像和3D模型，可以促进联合考虑这些模式的研究
    • 一种利用3D模型学习语义图像表示的对比学习方法
    • 将语义概念与图像和3D模型联系起来，即使是在从未见过的位置

2. Related Work

Joint analysis of 3D and language

  1. Chenet al 对ShapeNet数据集的文本和3D形状的联合嵌入，并演示了这些嵌入在文本到形状检索和文本到形状生成上( text-to-shape retrieval and text-to-shape generation)
  2. 由ShapeNet椅子组成的三个数据集，通过发音区别(utterances distinguishing)将其中的一把椅子与另外两把椅子区分
  3. Chen为室内场景构建多模态数据集，并使用自然语言对场景中的三维对象进行定位

Distilling information from Internet collections

  1. 一些作品利用Internet收藏通过图像聚类技术(image clustering techniques)，捕捉landmarks， events，或者named parts
  2. 另一些工作分析大规模旅游图像中的相机视点，自动总结场景(scene)或将其分割成组件(segment it into components)
  3. 3D维基百科分析旅游地标的文本描述，利用照片的共同出现来注释特定的3D模型，如 Pantheon

3. The WikiScenes Dataset

  1. 在ImageNET上训练一个7K object classes的large scalefine-grained classifier
  2. 候选集合中的每个盒子都被划分到7K class中的一个,根据包含类别名称和同义词的ImageNET 同义词集合，从预测对象类中提取单词包
  3. 然后，将单词包投影到向量空间，利用余弦距离与投影的查询文本进行匹配，得到 score

4.1 Object retrieval evaluation on ReferIt dataset

  1. ReferIt 数据集：

    • 包含图像区域的最大的公开可用数据集

    • 它包含了来自IAPR TC-12数据集的20,000张图像

    • 结合SAIAPR-12数据集的分割图像区域

    • 以及针对旨在从注释中识别图像区域的双人游戏中收集的图像区域的120K注释描述

    • 也包含一些模糊的(例如“anywhere”)和错误注释的例子，注释不对应任何对象

  2. 为了对该数据集进行评估，将2万张图像(以及它们的注释)分割为1万张用于训练和验证，1万张用于测试，并在所有带注释的图像区域上构造图像包围框描述元组作为训练实例

  3. trainval集中有59976个元组(image, bounding box, description)，测试集中有60105个元组

  4. 数据集的实验中，训练和评估时只使用标注区域的边界框，边界框由标注器点击对应的SAIAPR-12数据集中的分割区域获得

  5. 为了测试结合空间配置 Xspatial 和scene-level 上下文特征 xcontext 的效果，在ReferIt的微调中评估不同的设置

    • 通过在微调和测试期间将 Xspatial 和 Wglobal 设置为0
    • 该模型只能学习根据 candidate boxes中的局部图像描述符 Xbox为盒子打分
    • 类似地，通过将 Wglobal 设置为0，模型可以学习 Xbox和 Xspatial 上的评分函数，但不能利用 scene-level 上下文

  6. 作为对比，我们直接在ReferIt上训练SCRC模型，而没有对MSCOCO进行预先训练，在训练和测试时，将 Xspatial 和 Wglobal 设置为0，denoted by SCRC (w/o context, spatial, transfer)

  7. 模型中的CNN参数由VGG-16 net初始化，其他参数随机初始化

  8. 我们使用两个测试场景进行评估:

    检索到的区域与ground truth边界框至少有50%的IoU重叠，则被认为是正确的

    • 在第一个场景中，给定一个图像和一个文本查询，该模型被要求从该图像中所有带注释的区域中检索相应的图像区域
    • 在第二个场景中，给定文本查询，模型从对象建议方法产生的一组候选边界框中检索图像区域（这是一个更难的任务，但更接近真实的应用程序）

4.2 Object retrieval evaluation on Kitchen dataset

  1. Kitchen数据集：

    • 包含从ImageNET层次结构的Kitchen /household子树中采样的606张图像，每个图像有10个不同的描述注释

    • 由于该数据集中的对象几乎占据了整个图像，在候选对象建议框中不使用检索

  2. 在测试过程中，对于每个查询文本，候选集由11个具有ground truth和10个干扰物的图像组成。干扰物的采样要么来自同一个Kitchen数据集(“Kitchen”实验)，要么来自整个ImageNET(“ImageNET”实验)
  3. 为了在这个数据集上评估，将数据集分成两部分：以300幅图像作为trainval集，306幅图像作为测试集
  4. 与4.1节类似，首先在MSCOCO数据集上预训练SCRC模型用于image captioning，然后在trainval集合上对模型进行微调
  5. 型通过对ground truth和10个干扰物的候选集合进行 image-level 级检索来测试，使用从整个图像中提取的特征作为 Xbox
  6. 由于数据集不涉及 spatial configurations 或 scene-level 的上下文信息，在微调和测试期间，我们将 Xspatial 和 Wglobal 设置为0

5. Conclusion

  1. 本文提出了Spatial Context Recurrent ConvNet(SCRC)的自然语言对象检索方法。SCRC是一种基于局部图像描述符、空间配置和全局场景级上下文对候选框进行评分的recurrent neural network model。我们发现，将空间结构和全局上下文结合起来，可以显著提高自然语言对象检索的性能。在我们的方法中使用的循环网络模型导致了端到端可训练的评分函数

  2. 此外，我们还证明了自然语言对象检索可以通过预训练(pretraining)和自适应(adaptation)来传递在 image captioning 上学习到的知识

  3. 由于自然语言对象检索系统的困难之一是缺乏具有 object-level 注释的大型数据集，但利用带有image-level注释的数据集可以缓解这个问题，image-level 注释通常比 object-level 描述更容易收集
